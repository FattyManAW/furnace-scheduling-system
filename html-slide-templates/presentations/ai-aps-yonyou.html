用友 APS 产品架构

两层计划体系

三层 AI Agent 驱动的 APS 架构

第一层：AI Agent 前置算料

  • 中长期协作采购
  • APS 产能物料同步规划
  • 智能预测 + 原料需求预判

第二层：产线级别排程

  • APS 产线级计划
  • 基于约束规划（周/日计划）
  • 工序级作业指令下发

集团层中期计划（季/月）

  • 目标：做什么 · 做多少 · 在哪个厂做
  • 方法：全域优化算法 + 利润最优为目标
  • 约束：订单交期 + 各厂产能 + 原料成本 + 运输成本
  • 输出：各工厂生产任务建议 + 原料采购建议

工厂层短期计划（周/日）

  • 承接集团层生产任务 → 工序级指令
  • 考虑：设备状态 + 炉室利用最大化 + 工作时间 + 连批生产
  • 工厂层排程包含集团层 70% 的逻辑
  • 输出：可执行的工序级工单 + 派工指令

企业碳管理痛点分析

挑战

📊 数据分散

  • 数据来源分散，手工收集效率低
  • Excel 计算易出错，单位换算混乱
  • 缺乏统一数据平台，数据孤岛严重

⏱️ 周期长

  • 每年碳盘查需 1-3 个月
  • 数据反复核对，耗时耗力
  • 响应监管时效性差

📐 标准复杂

  • ISO 14064 / GHG Protocol 等标准繁多
  • 范围 1/2/3 分类核算要求高
  • 缺乏专业人才和经验积累

💡 价值难显现

  • 碳管理视为合规成本，难以体现商业价值
  • 减排机会识别困难
  • 缺少数据支撑碳资产变现
