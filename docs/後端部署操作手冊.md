# 後端部署操作手冊

**系統**：干式套管最佳化排爐系統 API v2.0.0  
**最後更新**：2026-05-19

---

## 1. 快速部署（Docker Compose）

### 前置條件

- Docker + Docker Compose 已安裝
- 從 GitHub 拉取原始碼：`git clone https://github.com/FattyManAW/furnace-scheduling-system.git`

### 一鍵啟動

```bash
cd furnace-scheduling-system
docker compose up -d
```

啟動後：
- API：`http://<server-ip>:8002`
- Swagger 文件：`http://<server-ip>:8002/docs`
- 健康檢查：`http://<server-ip>:8002/health`

### 查看日誌

```bash
docker compose logs -f api
```

### 重啟 / 關閉

```bash
docker compose restart api   # 重啟
docker compose down          # 關閉（保留資料庫）
docker compose down -v       # 關閉 + 清除資料庫（下次啟動會重新 seed）
```

### 設定 ALLOWED_ORIGINS（CORS）

若前端部署到正式網域，需設定 CORS：

```bash
# 使用 .env 檔案
echo 'ALLOWED_ORIGINS=https://fattymanaw.github.io,https://your-domain.com' > .env
docker compose up -d
```

## 2. 部署至 Railway

### 步驟

1. GitHub 連動 Railway → 選擇 repo `FattyManAW/furnace-scheduling-system`
2. Railway 自動偵測 Dockerfile（或手動設定）
3. 設定環境變數：
   - `PORT=8002`
   - `ALLOWED_ORIGINS=https://fattymanaw.github.io`
4. 掛 Volume（因 SQLite 需要持久化）：
   ```
   Mount Path: /app/backend
   ```
5. Deploy → Railway 自動取得 *.railway.app 域名

### 成本

- $5/mo（含 1GB volume + 0.5vCPU + 512MB RAM）

---

## 3. 部署至 Render

### 步驟

1. Render Dashboard → New Web Service → 連結 GitHub repo
2. Build Command: `docker build -t furnace-api .` 或直接 `pip install -r backend/requirements.txt`
3. Start Command: `cd backend && python3 -m uvicorn main:app --host 0.0.0.0 --port $PORT`
4. Disk Mount（免費層需要）：
   ```
   Mount Path: /app/backend
   Size: 1 GB
   ```
5. 環境變數：`PORT=8002`、`ALLOWED_ORIGINS=https://fattymanaw.github.io`

### 成本

- 免費層：每月 750h，閒置 15 分鐘後休眠（冷啟動 ~30s）
- $7/mo：零冷啟動

## 4. 部署至 Fly.io

### 步驟

```bash
# 安裝 flyctl
brew install flyctl

# 登入 + 初始化
flyctl auth login
flyctl launch

# 建立 volume（SQLite 持久化）
flyctl volumes create furnace_data --size 1

# 部署
flyctl deploy
```

### 成本

- $3/mo（1 shared CPU + 256MB RAM + 1GB volume）

## 5. 手動部署至 VPS（Ubuntu 22.04）

```bash
# 1. 安裝 Docker
sudo apt update && sudo apt install docker.io docker-compose-v2 -y
sudo usermod -aG docker $USER  # 重新登入生效

# 2. 拉取原始碼
git clone https://github.com/FattyManAW/furnace-scheduling-system.git
cd furnace-scheduling-system

# 3. 設定環境變數
echo 'ALLOWED_ORIGINS=https://fattymanaw.github.io' > .env

# 4. 啟動
docker compose up -d

# 5. 設定防火牆
sudo ufw allow 8002/tcp

# 6. （可選）Nginx 反向代理 + HTTPS
sudo apt install nginx certbot python3-certbot-nginx -y
# ... 設定 server block + certbot
```

## 7. 監控與維運

### 健康檢查

```bash
curl http://localhost:8002/health
# → {"status":"ok"}
```

### 資料備份

```bash
# SQLite 備份（Docker 內）
docker exec furnace-api cp /app/backend/furnace_schedule.db /app/backend/furnace_schedule.db.bak

# 複製到主機
docker cp furnace-api:/app/backend/furnace_schedule.db ./backup/
```

### 重新匯入種子資料

```bash
docker exec furnace-api python3 -c "
from database import SessionLocal
from seed_data import seed_all
db = SessionLocal()
seed_all(db)
db.close()
"
```

## 8. 前端連線設定

部署後端後，更新前端 `api.js` 指向正式 API 位址：

```js
// frontend/src/lib/api.js
const API_BASE = import.meta.env.VITE_API_URL || "http://localhost:8002";
```

若前端也部署到 GitHub Pages，需重新 build 並 push 到 `frontend-build` 分支：

```bash
cd frontend
VITE_API_URL=https://your-api-domain.com npm run build
# push dist/ 到 frontend-build branch
```
