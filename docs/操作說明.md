# 📖 操作說明

## 1. 系統啟動

### 後端
```bash
cd backend
pip install fastapi uvicorn sqlalchemy pydantic
python seed_data.py --init    # 首次啟動：初始化 DB + 匯入資料
python main.py                 # 啟動 API 伺服器（port 8002）
```

### 前端
```bash
cd frontend
npm install
npm run dev                    # 開發模式（port 5173）
npm run build                  # Production build
```

## 2. 頁面說明

### 儀表板（首頁）
- **總覽卡片**：訂單數、使用爐數、已排工時、模具型號數
- **工時使用率**：視覺化進度條，紅色 = 超過 95%
- **訂單狀態**：待排 / 已排 / 完成 / 逾期
- **合約分類**：依合約號統計待排訂單
- **逾期警示**：列出所有已逾期且未完成的訂單

### 訂單管理
- **列表**：分頁表格，支援搜尋（計劃單號 / 合約號）與狀態篩選
- **新增/編輯**：Modal 表單
- **批量匯入**：每行一個 JSON 格式的訂單資料
- **操作**：編輯、刪除、狀態變更

### 模具庫存
- **卡片檢視**：外徑 × 內徑 × 長度、存量
- **低庫存警示**：存量低於 10 支時顯示紅色警示
- **入庫/出庫**：快速調整存量
- **新增/編輯模具**：Modal 表單

### 排程設定
- **策略選擇**：交期優先 / 填滿優先 / 平衡模式
- **訂單選擇**：可篩選特定訂單或全選
- **執行排程**：點擊「開始排程」，系統自動計算最優分配
- **結果預覽**：
  - 已排入 / 未排入 / 總工時 / 使用爐數
  - 各爐排程詳情（可展開收合）
  - 警示訊息（超工時、無匹配爐位等）
  - 最近排程記錄

### 甘特圖
- **爐次為縱軸**：以乾燥罐為行
- **日期為橫軸**：30 天視窗，可前後翻頁
- **彩色條塊**：依合約號分色，懸浮顯示詳細資訊
- **週末標示**：紅色底色

### 報表匯出
- **訂單 CSV**：完整訂單資料
- **排程 CSV**：當前排程結果
- **訂單 JSON**：完整 JSON 格式
- **待排合約統計**：依合約號統計訂單數與總數量
- **逾期訂單一覽**

### 系統設定
- **一般設定**：每日工時上限、大產品門檻、低庫存 threshold
- **預設排程策略**：設定預設使用的排程策略
- **資料庫操作**：重新匯入、清除排程
- **通知設定**：工時超標、逾期、低庫存等通知開關

## 3. 業務規則

| 規則 | 說明 |
|------|------|
| 每日工時上限 | 1098 小時（全局，非 per-kiln）|
| 大產品限制 | 外徑 ≥ 470mm 只能進有大槽位的爐 |
| 排程順序 | 依交期日期由早到晚優先分配 |
| 爐位選擇 | 優先分配空閒槽位最多的爐 |
| 製程工時 | 依電壓等級（10/24/36/40kV）查詢 process_hours |
